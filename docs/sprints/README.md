# Спринты RAG v2

## Для вас (минимум действий)

**Новый чат:** напишите что угодно из разряда «продолжай RAG», «следующий спринт», «делай» — или просто опишите проблему. Номера спринтов и copy-paste **не нужны**.

Агент сам:
1. Читает [`SPRINT_STATE.md`](./SPRINT_STATE.md) — где остановились
2. Читает [`RAG_MIGRATION_PLAN.md`](../architecture/RAG_MIGRATION_PLAN.md) §6 — задачи и DoD текущего спринта
3. Сверяет с §11 — архитектурные решения
4. Смотрит `git log` — что уже в коде

**Вы отвечаете только когда:**
- агент нашёл **противоречие** с планом или §11;
- нужно **архитектурное решение**, не описанное в плане;
- агент предлагает **отклониться** от плана (scope change).

Во всех остальных случаях — не беспокоить.

## Два файла вместо десятка

| Файл | Кто ведёт | Содержание |
|------|-----------|------------|
| [`RAG_MIGRATION_PLAN.md`](../architecture/RAG_MIGRATION_PLAN.md) | Архитектор (редко) | Полный план: задачи S0-1…S10-5, DoD, NFR, §11 |
| [`SPRINT_STATE.md`](./SPRINT_STATE.md) | Агент (каждый спринт) | Текущий спринт, прогресс задач, env, журнал итогов |

Подробности задач **не дублируются** — они только в плане §6.

## Что агент делает в конце спринта

1. Тесты по задачам спринта
2. Обновляет `SPRINT_STATE.md` (статус задач, журнал, env)
3. `git commit` + `push`
4. Краткий отчёт вам — без вопросов, если всё по плану

## Карта спринтов (справка)

См. [`RAG_MIGRATION_PLAN.md` §6](../architecture/RAG_MIGRATION_PLAN.md): S0…S10 (+ S7b).
